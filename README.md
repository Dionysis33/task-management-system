 Περιγραφή Έργου
Σχεδιασμός και υλοποίηση ενός συστήματος διαχείρισης εργασιών (To-Do) για φοιτητές, με στόχο την αποτελεσματική οργάνωση ακαδημαϊκών και προσωπικών υποχρεώσεων. Η εφαρμογή έχει αναπτυχθεί με έμφαση στην επεκτασιμότητα, την ασφάλεια και την υψηλή απόδοση.

 Τεχνολογικό Υπόβαθρο
Java 21: Αξιοποίηση των Virtual Threads για τη βελτιστοποίηση της διαχείρισης πόρων και την αύξηση της απόδοσης.

Spring Boot 3.3.0: Χρήση του πλέον σύγχρονου framework για σταθερότητα και πλήρη υποστήριξη.

Spring Security 6: Ολοκληρωμένη υλοποίηση ελέγχου πρόσβασης (RBAC) και προστασία ευαίσθητων δεδομένων.

MySQL: Σύστημα βάσης δεδομένων για τη διασφάλιση της ακεραιότητας και της συνεκτικότητας των δεδομένων.

 Αρχιτεκτονική & Σχεδιασμός
Το σύστημα υιοθετεί την Πολυεπίπεδη Αρχιτεκτονική (N-Tier) για τον αυστηρό διαχωρισμό ευθυνών (Separation of Concerns):

4.1 Επίπεδα Εφαρμογής
Controller Layer: Διαχείριση αιτημάτων HTTP, επικύρωση δεδομένων και δρομολόγηση.

Service Layer: Κεντρικό επίπεδο όπου ενσωματώνεται όλη η επιχειρησιακή λογική και οι κανόνες του συστήματος.

Repository Layer: Χρήση του Repository Pattern για ανεξάρτητη και ασφαλή επικοινωνία με τη βάση δεδομένων.

4.2 Μοτίβα Ασφάλειας
DTO Pattern: Χρήση ειδικών αντικειμένων μεταφοράς δεδομένων για την πλήρη αποφυγή επιθέσεων Mass Assignment (Over-posting).

 Κύριες Λειτουργίες
User Management: Σύστημα εγγραφής και αυθεντικοποίησης με χρήση κρυπτογραφημένων κωδικών πρόσβασης.

RBAC: Έλεγχος πρόσβασης βάσει ρόλων, με σαφή διαχωρισμό δικαιωμάτων μεταξύ Διαχειριστών (Admin) και Μελών (Member).

Task CRUD: Πλήρης κύκλος διαχείρισης εργασιών (Δημιουργία, Προβολή, Ενημέρωση, Διαγραφή).

Filtering & Search: Δυνατότητα αναζήτησης και φιλτραρίσματος εργασιών βάσει κατηγορίας και ημερομηνίας λήξης.
