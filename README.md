 Περιγραφή Έργου
Σχεδιασμός και υλοποίηση ενός συστήματος διαχείρισης εργασιών (To-Do) για φοιτητές, με στόχο την αποτελεσματική οργάνωση ακαδημαϊκών και προσωπικών υποχρεώσεων. 
Η εφαρμογή έχει αναπτυχθεί με έμφαση στην επεκτασιμότητα, την ασφάλεια και την υψηλή απόδοση.


 Τεχνολογικό Υπόβαθρο

Java 21: Αξιοποίηση των Virtual Threads για τη βελτίωση της απόδοσης και της διαχείρισης πόρων.

Spring Boot 3.3.0: Χρήση του πιο σύγχρονου framework για σταθερότητα και επίσημη υποστήριξη.

Spring Security 6: Υλοποίηση ελέγχου πρόσβασης βάσει ρόλων (RBAC) και προστασία δεδομένων.

MySQL: Σύστημα διαχείρισης βάσης δεδομένων για τη διασφάλιση της συνεκτικότητας των δεδομένων.

 Αρχιτεκτονική & Σχεδιασμός
Το σύστημα ακολουθεί την Πολυεπίπεδη Αρχιτεκτονική (N-Tier) για τον αυστηρό διαχωρισμό ευθυνών:

Controller Layer: Διαχείριση αιτημάτων HTTP και δρομολόγηση.

Service Layer: Ενσωμάτωση επιχειρησιακής λογικής και κανόνων.

Repository Layer (Pattern): Ανεξάρτητη επικοινωνία με τη βάση δεδομένων.

DTO Pattern: Χρήση Data Transfer Objects για την αποφυγή επιθέσεων Mass Assignment (Over-posting).


 Κύριες Λειτουργίες

User Management: Εγγραφή και αυθεντικοποίηση χρηστών με κρυπτογραφημένους κωδικούς.

RBAC: Διαχωρισμός δικαιωμάτων μεταξύ Διαχειριστών (Admin) και Μελών (Member).

Task CRUD: Πλήρης διαχείριση εργασιών (Δημιουργία, Ανάγνωση, Ενημέρωση, Διαγραφή).

Filtering & Search: Αναζήτηση εργασιών βάσει κατηγορίας και ημερομηνίας λήξης.
